The Discord click-paths and feature behavior below match Discord's official support articles as of 2026-06-18, but Discord changes its interface often: if a button is named differently on your screen, look for the nearest equivalent inside Server Settings. Membership terms, the name/likeness release, and the content-usage license that creators agree to live in the separate Sensitive Room Terms and Privacy drafts; have qualified counsel review those before anyone signs or joins, with specific attention to likeness/publicity rights, content licensing, and any participant under 18.

This guide assumes you have never opened Discord. It walks you click-by-click from making an account to running the room day to day. Do the setup on a computer. Read section 12 first if you only have 30 minutes and need to go live today.

01 · What Discord is & why we use it

The room behind the workshop

Discord is a free app for running a private community. Think of it as a group chat that is organized into labeled rooms instead of one long thread, plus voice rooms you can drop into like a phone call. People join through one invite link and everything lives in one place.

Chapter 01 is Thursday, June 25, 2026 (7PM workshop, then a 10PM World Cup watch party) at Haus of K, 345 Adams St, Brooklyn. Keep the room's context current to that night.

We use it for three jobs, and only these three at the pilot:

  • Community. Accepted creators meet each other, introduce themselves, share wins, and stay in the network after the workshop day. This is what keeps a chapter alive into the next one.
  • Organized conversations. The chapter brief, the brand briefs, logistics, and Q&A each get their own channel, so nothing important gets buried in chat.
  • Content-submission intake. Each accepted creator owes 3 organic pieces. They post the live links in one channel and Sensitive Room logs each one to the tracker. This is how we prove delivery to sponsoring brands.

Quick glossary (read once, refer back)

Server
Your whole community space. Shows as an icon down the left edge. Ours will be named "Sensitive Room."
Channel
One room inside the server. Text channels (#like-this) are for typing; voice channels are for talking.
Category
A folder that groups related channels (for example an INFO folder holding several channels).
Role
A label you attach to a person, like @Roster or @Brand. Roles grant permissions and decide who can see which channels. This is the single most important concept for keeping deal channels private.
Thread
A side-conversation that branches off one message, so a long discussion does not flood the main channel.
DM
Direct message. A private one-to-one chat outside any server.
Voice channel
A room you click into to talk live, share screen, or host office hours. No call to schedule; people just join.
Onboarding
Discord's new-member flow. It greets people, asks a couple of questions, and routes them to the right channels when they first join.
AutoMod
Discord's built-in auto-moderation. It can block messages that contain banned words or spam before anyone sees them.
02 · Account + app setup

Make your account first

You can register from a browser, the desktop app, or the phone app. The fastest path is a browser on your computer, because you will do all the admin work on desktop anyway (some admin features do not exist on mobile).

Desktop (browser or app)

  1. Go to discord.com and click Login, then Register. (Or download the desktop app and register inside it.)
  2. Enter your Email, then a Display Name (what people see), a Username (your unique handle), a Password, and your Date of Birth.
  3. Solve the human-verification box if it appears, then submit.
  4. Open the email Discord sends you and click Verify Email. Do this now, it is required before you can turn on Community in section 4.

Mobile (iOS / Android app)

  1. Install Discord from the App Store or Google Play and open it.
  2. Tap Register, enter your phone number or email, then set Display Name, Username, Password, Date of Birth.
  3. Verify through the email link (or the SMS code if you registered by phone).
Do this onceTurn on two-factor authentication (2FA) under User Settings > My Account. The owner account controls everything, so protect it. Discord's moderator features also expect a secured owner account.
03 · Create the server

Build "Sensitive Room"

Down the left sidebar there is a green + button labeled "Add a Server." Everything below starts there.

Option A · From scratch (recommended for a first-timer)

  1. Click the green + "Add a Server" icon in the left sidebar.
  2. In the pop-up choose Create My Own (a blank slate), pick a purpose if it asks.
  3. Enter the Server Name Sensitive Room and optionally upload a server icon (the room.png or poster art in this folder works).
  4. Click Create. Your server now exists with one #general text channel, one General voice channel, and one role, @everyone. You will build the rest in section 5.

Option B · From a template (faster, one extra step)

  1. Click the green + "Add a Server" icon, then choose a template (or paste a template link). You can preview it, rename it to Sensitive Room, and add an icon before creating.
  2. A template clones categories, channels, roles, and permissions, so it is a head start.
Template caveatCreating a server FROM a template turns Community OFF and strips the Community-only features. Even if a template looks "community-ready," you must still run section 4 (Enable Community) afterward. For your first build, Option A is simpler to reason about.

Later, once your server is shaped the way you like, you can save it as your own reusable template via Server Settings > Server Templates > Generate Template (desktop only, needs Manage Server) so Chapter 02 can clone it.

Set up on desktopDo all the configuration on a computer. Community Onboarding in particular can only be set up on desktop or browser, not the phone app. Members can join and use the server from any device, it is only the admin setup that needs a computer.
04 · Turn on Community features

Why this matters

Enabling "Community" is the switch that unlocks the cohort-grade tools we need: Onboarding, the Welcome Screen / Server Guide, Rules Screening (members must agree to rules before they can talk), AutoMod, and Announcement channels. Without it, the server is just a basic chat. Turn it on before you invite anyone.

  1. Open Server Settings (click the server name at top-left, then Server Settings).
  2. Find Enable Community and click Get Started, then follow the wizard.
  3. The wizard enforces two safety defaults: members must have a Verified Email before they can talk or DM, and it turns on media scanning for all members. Leave both on.
  4. Set the default notification setting to Only @mentions so a 50-person cohort is not buried in pings.
  5. The wizard asks you to designate a Rules channel and a Community Updates channel (where Discord sends you admin and moderation notices). Pick or create those, then finish.
Do not confuse this with Discovery"Enable Community" is what you want. "Enable Server Discovery" is a different, optional thing that publicly lists your server and needs roughly 1,000+ members and 8 weeks of age. We are a private, invite-only room, so never turn on Discovery.
05 · Channel map

The structure to build

Keep it lean for the pilot. The only public-facing layer is INFO; everything of value sits behind the @Roster role. The grouping below also satisfies Onboarding's rule that you need at least 7 default channels with at least 5 public ones (see section 7).

  • INFO Public
    • #welcome · what this room is, in two lines
    • #rules · the guidelines members must agree to (Rules Screening channel)
    • #start-here · how to get your @Roster role and unlock the room
    • #announcements · Announcement channel; admins post, members read
  • COMMUNITY Roster
    • #introductions · everyone says who they are when they join
    • #general · the main hangout chat for the cohort
    • #wins · post published work and results
    • #ask-anything · open Q&A for the room and the team
  • CHAPTER 01 Roster
    • #chapter-01-brief · date, address (345 Adams St, Brooklyn), schedule, the assignment
    • #content-submissions · drop your organic video links here, one per assigned brand, up to 3 (see section 9)
    • #brand-briefs · the sponsor brief, do's and don'ts, required tags and disclosure
  • DEALS Roster + Admin
    • #open-opportunities · paid collabs and briefs posted by the team (gated)
    • #deal-room · where a selected creator and the team work out a specific paid deal (gated)
  • VOICE Roster
    • Lounge · drop-in voice room for the cohort
    • Office Hours · where Daesung and the team go live for feedback sessions
  • MOD-ONLY Admin + Host
    • #mod-chat · team back-channel
    • #automod-alerts · where AutoMod posts its flags (set in section 10)

To add a category: hover the channel sidebar and click + > Create Category. To add a channel: hover a category, click its +, choose Text / Voice / Announcement, name it, create.

06 · Roles & permissions

Who sees what

A role is a label on a person. A channel can be set so only certain roles see it. That is how #deal-room and #open-opportunities stay private. Create roles at Server Settings > Roles > Create Role. New roles drop in just above @everyone; drag to reorder. Give each its own color and turn on "Display role members separately" so @Roster shows as its own group in the member list.

RoleWhoCan seeKey permissions
@AdminAlex, Justin, AramEverything, including MOD-ONLY and DEALSAdministrator
@HostDaesungAll cohort + DEALS + MOD-ONLYManage Messages, Mute/Move in voice, Timeout; not Administrator
@BrandSponsor repsINFO + their #brand-briefs only; not DEALS, not other chaptersRead/send in their brief channel only
@RosterAccepted creatorsINFO + COMMUNITY + CHAPTER 01 + DEALS + VOICEStandard send/react; no manage
@AlumniPast-chapter creatorsINFO + COMMUNITY (kept in the network); not current chapter or DEALSStandard send/react
@GuestOptional visitorsINFO onlyRead; limited send
@everyoneBuilt-in baselineINFO only; denied on all private categoriesMinimal

How to make a channel private (permission overrides, in plain terms)

A "permission override" is a per-channel rule that overrides the server-wide setting. You set it as a green check (allow) or a red X (deny).

  1. The smart move: set the override on the CATEGORY, not each channel. Hover the category > gear (Edit) > Permissions.
  2. Select @everyone and set View Channel to the red X (deny).
  3. Click +, add @Roster (and @Admin / @Host), set their View Channel to the green check (allow).
  4. Have each channel inside sync to the category so it inherits the same Roster-only access automatically. Now any new channel you drop in that category is private by default.
The one rule beginners get wrongChannel permissions do not obey role hierarchy. If a person has any role that ALLOWS "View Channel," they can see the channel, even if another of their roles denies it. Any single allow wins. So never give @Guest or a broad role an "allow" on a private category. After setup, test with Server Settings > Roles > View Server As Role to confirm a plain member cannot see DEALS.

Hierarchy still matters for managing people: you can only manage roles below your own and only grant permissions you already have. So keep @Admin at the top.

07 · Onboarding & verification

Greet new members and keep spam out

Verification level (anti-spam gate)

At Server Settings > Safety Setup > Verification Level, pick how much a new account must clear before it can chat:

LevelRequirement
NoneChats immediately
LowVerified email
MediumVerified email + account at least 5 minutes old
HighThe above + a member of this server for at least 10 minutes before chatting
HighestVerified phone number on their account

For an invite-only creator room, set Medium or High. A real invited creator will not mind a 10-minute wait, and it deters throwaway accounts.

Rules Screening (must agree before participating)

  1. Go to Server Settings > Safety Setup > Server Rules.
  2. Write the rules (or start from Discord's examples), then save. New members must explicitly agree before they can talk, react, or DM. Use the same #rules channel from your map.

There is also an optional Apply to Join (Server Member Applications) feature, a form with custom questions before someone can even enter. Useful later if you want to vet creators by application. For Chapter 01, single-use invites (section 8) plus assigning @Roster by hand is enough.

Community Onboarding (the welcome flow)

At Server Settings > Onboarding (desktop only, needs Manage Server + Manage Roles). Requirements: pick at least 7 Default Channels, of which at least 5 let @everyone View and Send. Your INFO + COMMUNITY channels satisfy this; keep CHAPTER 01, DEALS, and VOICE out of the public defaults.

  • Default Channels · what a new member sees on joining. Put the public INFO and COMMUNITY channels here.
  • Questions · short customization questions; an answer can grant extra channels or roles. Do not auto-grant @Roster here, that one you assign by hand after you confirm the person is accepted.
  • Server Guide · a short welcome message and a to-do list for newcomers. Preview it as a new member, then Enable.

AutoMod basics

At Server Settings > AutoMod, turn on at least the "Commonly Flagged Words" ready-made list and a "Custom Keyword" rule for anything you want blocked. For each rule, choose to Block the message and send an Alert to #automod-alerts. Admins and anyone with Manage Server are always exempt, so the team is never filtered. More in section 10.

08 · Getting creators in

Invite, then assign the role

Create a controlled invite link

  1. Hover a channel (use #welcome) and click Create Invite, or the invite icon.
  2. Click Edit invite link. Set Expire after to 7 days and Max number of uses to 1 for a single creator, so a link cannot be passed around.
  3. Click Generate a New Link and send it to that creator (paste it into the acceptance email or form).
  4. Revoke or manage any old links later at Server Settings > Invites.

For a general cohort blast, you can instead make one link with a higher max-uses cap (for example 30) and a short expiry. Single-use per person is tighter and is the better default for a curated roster.

Assign @Roster when an accepted creator joins

  1. Quick way: in the member list, right-click the person > Roles > check Roster (and a chapter role if you use one).
  2. Or use Server Settings > Members, find the person, click + under Roles, check the role. The Members page also lets you filter by role to audit exactly who has @Roster access.
  3. You need Manage Roles to do this, and you can only assign roles below your own, so do it from an @Admin or @Host account.
09 · The content-submissions workflow

How the organic videos get tracked

Each accepted creator makes one organic video for each sponsoring brand assigned to them, capped at 3 total (this is a condition of acceptance): 1 brand means 1 video, 2 brands means 2, 3 brands means 3, and if 4 to 10 brands sponsor the chapter, Sensitive Room picks 3 brands per creator and they make one video each. When there is more than one brand, Sensitive Room assigns which brands each creator covers so coverage stays fair. Creators post the live links in #content-submissions, and Sensitive Room logs each one to the tracker. Pin the required format at the top of the channel so every submission is uniform.

Pin this in #content-submissions SUBMISSION FORMAT (copy this, fill it in, one message per video):
Brand tagged: @brandhandle + @sensitiveones.agency
Link: (paste the live Reel / TikTok URL)
Date published: YYYY-MM-DD
FTC disclosed: YES (used #ad / #sponsored or the Paid-partnership label)
Video: 1 of 3 / 2 of 3 / 3 of 3 (or 1 of 1, etc., to match your assigned brand count)

What counts (per the deal terms)

  • One in-feed short-form video per assigned brand (Instagram Reel or TikTok is fine).
  • Must tag the brand handle and @sensitiveones.agency.
  • Must carry a clear FTC material-connection disclosure: #ad or #sponsored or the platform's Paid-partnership label.
  • Timing: first video within 14 days of the workshop; all videos within 30 days.

How the team logs it

  1. When a submission posts, react with a check emoji so the creator knows it landed and so you can see at a glance what is logged.
  2. Open the tracker (Sheet / Notion) and add a row: creator, brand assigned, piece number, link, date, FTC = yes, logged-by.
  3. Reply in a thread on their message if anything is missing (no tag, no disclosure) so the fix stays attached to the original post.
  4. After a creator's assigned videos are all in and the 30-day window closes, Sensitive Room reviews who over-delivered and shortlists creators to link with brands for paid work.
Why the format is strictThe tag and the FTC disclosure are not optional polish. The @sensitiveones.agency tag is how we prove coverage to sponsors, and the disclosure is a legal requirement on sponsored content. A piece missing either one does not count toward the creator's 3.
10 · Moderation & safety

Keep it clean and calm

  • AutoMod. At Server Settings > AutoMod, keep the Commonly Flagged Words list on, add a Custom Keyword rule for anything specific, and turn on spam and mention-spam limits. Route alerts to #automod-alerts. You can also block banned words from being used in nicknames.
  • Reporting. Anyone can right-click a message > Report it to Discord. For in-house issues, members can ping @Host or @Admin or post in #ask-anything.
  • Timeout. The soft tool. Right-click a member > Timeout to mute them for a set period without removing them. Use this first for heat-of-the-moment problems.
  • Kick. Removes a person; they can rejoin with a new invite. Use for a one-off who does not belong.
  • Ban. Removes and blocks the account from returning. Use only for bad actors.
  • Stay organized. Turn on Slow Mode in #content-submissions and #open-opportunities so they stay clean lists, not chat. Pin the important post at the top of each channel. Archive old chapter channels rather than deleting them.
Add a second moderatorBefore you open the doors, give @Host (Daesung) or one trusted teammate the moderator permissions (Manage Messages, Timeout, Kick, not Administrator). One person should not be the only one who can calm a channel.

12 · First 30 minutes checklist

The exact minimum to go live

If you do only these, in order, the room is ready for your first creator. Everything else can be added later.

  1. Create the account and verify your email (section 2).
  2. Turn on 2FA on your account.
  3. Create the server, named Sensitive Room (section 3, Option A).
  4. Run Enable Community and finish the wizard; set default notifications to Only @mentions (section 4).
  5. Create the categories INFO, COMMUNITY, CHAPTER 01, DEALS (section 5). VOICE and MOD-ONLY can wait.
  6. Add the core channels: #welcome, #rules, #start-here, #announcements, #introductions, #general, #content-submissions, #open-opportunities.
  7. Create the @Roster role (section 6).
  8. On the DEALS category, deny @everyone View and allow @Roster; sync its channels (section 6).
  9. Set Verification Level to Medium or High and turn on Rules Screening with a few rules (section 7).
  10. Write the welcome message in #welcome and pin the submission format in #content-submissions (section 9).
  11. Test with View Server As @everyone: confirm DEALS is hidden and INFO is visible.
  12. Create a single-use, 7-day invite and send it to your first creator (section 8).
